Doc Tool Documentation

This is the documentation for the doc tools themselves. User-visible documentation is in other subdirectories of the docs/src directory, and in the the docs directory.

See here for a list of tags available to mark up doc sources.

Building the Docs

In docs/src:

ant
This builds all the docs (html versions for the product only).
ant xxx
Where xxs is one of dguide, reference, or tutorials.
ant clean
Cleans everything.
ant clean-xxx
Where xxx is one of dguide, reference, or tutorials.
ant old-dguide, ant old-tutorials
Builds the non-docbook versions of the dguide and tutorials, into docs/old-dguide and docs/old-tutorials. Deprecated.
ant fast-dguide
Builds a fast version of the dguide, with broken cross-references (marked as "???"), into docs/fast-dguide. This is actually slower to build the first time, but faster after that if you only change a file or two. Deprecated unless you tell me (Oliver) that you find it useful.

The display of <todo> and <fixme> items is controlled by the ${LPS_HOME}/build.properties "release" property controls. If this property has a value of Release, neither todos nor fixmes are displayed. If it has a value of Internal, both todos and fixmes are displayed. If it has any other value, todos are displayed but fixmes are not. This is intended for alpha and beta releases.

Doc Sources

The dguide sources are in docs/src/dguide. Tutorials sources are in docs/src/tutorials. Reference sources are in docs/src/wrappers, docs/src/hardpages, the schema, the components sources, and the LFC sources.

Markup

Doc source pages are in XML. They use a tagset similar to (X)HTML, with extensions for language-specific constructs such as tagname, method, etc. The dguide and tutorials use an additional set of tags for cross-references, index entries, and other "book"-type markup. See here for additional information about the tag set available.
