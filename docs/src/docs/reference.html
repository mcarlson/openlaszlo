LZX Reference Internal Documentation

Table of Contents Generation

The table of contents is created by make-toc.toc.xsl. This file contains a list of hardpages, plus commands to include generate a chapter for each category (below).

Categories

Categories are used in the "Contents" tab of the TOC. The *.py tools create a file named index-categories.html. The make-toc.xsl stylesheet turns this into toc.xml, which is placed in docs/reference.

The list of categories is in categories.txt. Categories are documented in hardpages/info-categories.html.

The category of a page is specified as the content of the <lzcategory> tag in its <lzelement> header. These transformations are applied to it:

  1. If the <lztier> is "Base Component", the category is replaced by "Base Class" instead. (HTMLWrapper.py implements this. TODO: Edit the wrapper pages instead.)
  2. If the <lztier> is "Lz Component", the category is replaced by "Components".
  3. The "Views" category is replaced by "View Basics" (makeHTML.py).
  4. Singular category names are made plural. (makeHTML.py implements this.)
  5. Redmond classes are removed (make-toc.xsl).

nav.lzo

The nav.lzx app needs to be optimized. It should be precompiled standalone, i.e., by loading nax.lzx and optimizing it. Be sure not to optimize it in the lxz-reference frame as this will cause it to cache the target parameter and it will not work properly with the web docs.
