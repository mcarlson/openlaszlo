Building the Web Docs

This part of the build is in limbo. Web docs are currently the same as product docs.

Workflow

Use ant -f build-web.xml web web-zip to create a zip file. Mail it to Antun. He will install it on the staging server, where it can be tested.

Build

These tasks require the installation of these jars in the ant/lib directory. These jars aren't yet checked into perforce because we haven't yet verified that their presence doesn't disrupt the main build.

  • Styler
  • JTidy
  • Bean Scripting Framework
  • Rhino
ant -f build-web.xml web
Builds the web docs into build/web. This will produce a lot of warnings.
ant -f build-web.xml web-zip
Places the text files from the web build in web-docs.zip. This is what can be mailed to Antun for installation on the staging server. (He can't build on his machine, and the zip that includes the art assets is too large to mail.)
ant -f build-web.xml web-clean
Clean the web build files.

Specification

  • Rename *.html files to *.php
  • Replace HTML header and footer by php.
  • Fix internal links:
    • *.html to *.php
    • Tutorials directory is a sibling of lzx-reference, not of its parent. (This is now true in the product as well.)
    • laszlo-in-ten-minutes is relative to /lps/.
    • Include files (embed, images) are in a different location.
