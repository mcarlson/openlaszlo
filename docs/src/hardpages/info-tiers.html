Tiers

LZX is divided into tiers ranging from the very core features to the component sets. In general, elements in higher tiers are built using elements from the lower tiers.







Language Tags

These tags define the structure of a Laszlo application. The canvas, library, and include tags define the structure of a Laszlo application. The font and resource tags defines resources that are included in the application and are available as view attributes (the font and resource attributes, respectively). The class, attribute, and method tags are used to define and customize tags, and the attribute and method tags are also used to customize the appearance and behavior of instances that are created from both builtin and custom tags.

Tags in this tier don't have a corresponding JavaScript class.







Laszlo Foundation Classes

These are the names of classes that are built into the client framework. Names that begin with Lz are only available from script code. Other classes are available both as tag names, without the Lz prefix, and as JavaScript class names, with the Lz prefix. (These names are shown without the prefix.)







Text Markup

Tags that mark up HTML text. These tags may be used within a text element, tags that extend this tag, and tags whose definition includes a definition for <attribute name="text" type="html"/>. (See the documentation on Text Classes for more information about this last category.)

The text markup classes are:

a
Encloses hypertext link.
b
Bold character markup.
br
A line break.
font
Sets the font, fontface, and color of its contents
i
Italic character markup.
p
Encloses a paragraph.
pre
Encloses preformatted text.
u
Underlined character markup.






Laszlo Components

Components in the Laszlo components library. These tags can be used to build to web applications.

An application can include the entire set of Laszlo components by including the line <include href="base"/>. An application can include individual Laszlo components by including just the files that define those components. The Laszlo components are included automatically if an application refers to them but does not define or include them; see here for a discussion of implicit inclusion.







Base Components

Components in the Base components library. These tags can be used to build custom components using the class tag.

An application can include the entire set of Base components by including the line <include href="base"/>. An application can include individual Base components by including just the files that define those components. The Base components are included automatically if an application refers to them but does not define or include them; see here for a discussion of implicit inclusion.

The Laszlo component set extends the Base components with a specific look and feel.







Utility Components

Components in the Utility component set. These include custom layouts, and tags that add resizing and dragging behavior to components.

An application can include the entire set of Utility components by including the line <include href="utils"/>. An application can include individual Utility components by including just the files that define those components. The Utility components are included automatically if an application refers to them but does not define or include them; see here for a discussion of implicit inclusion.







Redmond Components

Note: The Redmond library is deprecated. These components will be removed from a future release.

Components in the Redmond library. These are the window tag, and standard controls such as button and checkbox, with a familiar appearance. These can be included by including the line <include href="redmond"/> in an application.







Development Components

Components intended for use during application development. These include tags for writing unit tests, which can be included by including <include href="lzunit"/> in an application. Another development component, the debugger, is not listed here but can be included by including a debug attribute on the canvas tag.







Services

Runtime services available to JavaScript code. Services are always available by their api name, and do not get instantiated, e.g.

    LzBrowser.loadURL( 'http://www.laszlosystems.com' );
